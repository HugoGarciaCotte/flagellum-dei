

## Plan: PWA + Offline-First Architecture with Capacitor Readiness

### 1. PWA Setup (vite-plugin-pwa)

Install `vite-plugin-pwa` and configure in `vite.config.ts`:
- Service worker with `navigateFallbackDenylist: [/^\/~oauth/]`
- Manifest with Quest Scroll branding (dark theme, gold icon colors)
- `runtimeCaching` strategy for Supabase API calls (NetworkFirst with fallback)
- Cache scenario text data aggressively (CacheFirst after initial fetch)
- Cache images with NetworkFirst — graceful fallback if offline

Add PWA meta tags and icons to `index.html`.

### 2. Offline Scenario Prefetch

Create `src/hooks/useOfflineScenarios.ts`:
- On app load (Dashboard mount), fetch all scenarios + their scenario_sections text
- Store in `localStorage` or IndexedDB (localStorage is simpler for text-only data)
- Provide a React Query `queryFn` wrapper that checks local cache when offline (`navigator.onLine`)

Modify existing queries:
- **Dashboard**: Wrap scenario fetch to read from cache when offline
- **HostGame / PlayGame**: Wrap section fetch to read from cache when offline
- Images (background_color is just a hex string currently, not actual images — so this is already offline-safe for text)

### 3. Offline-Aware Data Layer

Create `src/lib/offlineStorage.ts`:
- `cacheScenarios(scenarios, sections)` — saves to localStorage
- `getCachedScenarios()` / `getCachedSections(scenarioId)` — reads from cache
- `isOffline()` — checks `navigator.onLine`

### 4. Capacitor Readiness

No Capacitor installation now (user said "later"), but structure the app to be ready:
- Keep all routing client-side (already done with react-router)
- PWA manifest serves as the bridge
- No server-side rendering dependencies
- Create `/install` page with PWA install prompt instructions

### 5. Network Status Indicator

Add a small offline banner component (`src/components/OfflineBanner.tsx`) that appears when `navigator.onLine` is false, informing users they're in offline mode.
