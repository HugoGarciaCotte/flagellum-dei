

## Add Prerequisites as a Parseable Field

### What Changes

Prerequisites (`feat_prerequisites`) becomes a new embedded parseable field like `feat_one_liner`, `feat_specialities`, etc. It gets parsed from `<!--@ feat_prerequisites: ... @-->` tags, displayed in the admin panel, generated by AI, and checked by AI.

### Files to Change

#### 1. `src/lib/parseEmbeddedFeatMeta.ts`
- Add `prerequisites: string | null` to `EmbeddedFeatMeta` interface
- Parse `feat_prerequisites` key in `parseEmbeddedFeatMeta()`
- Generate `<!--@ feat_prerequisites: ... @-->` line in `generateParseableBlock()`

#### 2. `src/components/ManageFeats.tsx`
- Add a `hasPrerequisites` helper (same pattern as `hasSubfeats`, `hasSpecialities`)
- Add a prerequisites indicator icon in the feat row (e.g. `ShieldCheck` or `ListChecks` icon)
- Display prerequisites value in expanded feat view

#### 3. `src/components/FeatDetailsDisplay.tsx`
- Update to also read prerequisites from `parseEmbeddedFeatMeta()` in addition to the wiki template parser (`parseFeatFields`), preferring the embedded tag if present

#### 4. `supabase/functions/regenerate-description/index.ts`
- Add `prerequisites` to the `EmbeddedFeatMeta` interface
- Update `parseEmbeddedFeatMeta` and `generateParseableBlock` (duplicated in edge function)
- Add a `generatePrerequisites()` AI function that extracts prerequisite text from wiki content
- Call it alongside description/subfeats/specialities in `regenerate_all`

#### 5. `supabase/functions/check-feats-ai/index.ts`
- Add `prerequisites` to the `EmbeddedFeatMeta` interface and parser/generator (duplicated)
- Add `prerequisites` to the AI review prompt fields list
- Add `prerequisites` to the tool call schema so the AI can suggest add/modify/delete for it

#### 6. `supabase/functions/validate-feat/index.ts`
- Use `parseEmbeddedFeatMeta(feat.content)` to extract prerequisites and include them prominently in the AI validation prompt (so the validator doesn't have to guess from raw wiki content)

#### 7. `src/components/WikiLinkedText.tsx` & `src/components/WikiSectionTree.tsx`
- Read prerequisites from embedded meta as fallback/override when displaying feat hover/details

### Tag Format
```
<!--@ feat_prerequisites: [Feat X], Level 3 @-->
```
Free-form text string, same as `feat_one_liner`.
